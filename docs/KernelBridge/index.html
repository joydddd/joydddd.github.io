Research Initiative

KernelBridge

Expert GPU Performance for AI DSLs.

The Vision

Our vision is to let AI developers focus on models and algorithms while our compiler and LLM-assisted tools bridge their DSLs to expert-level GPU kernel implementations.

The Problem (The Gap)

Today, realized GPU efficiency depends on tight alignment across algorithm design, compiler/kernel design, and hardware execution. Most teams face a performance lottery because those layers are disconnected.

Practitioner Level

High-level DSLs (Triton, Helion, PyTorch).

Fast model iteration, but limited access to expert-tuned kernels and hardware-level controls.

The Bridge

Connects algorithm intent to compiler optimizations and scheduling choices, enabling collaborative kernel authoring across DSL boundaries.

Hardware Level

Expert-level performance (NVIDIA CuTe, CUDA).

Peak throughput depends on memory hierarchy and architecture-specific tuning that is often out of reach for application teams.

Collaboration Gap

Systems experts and algorithm researchers work in separate DSL ecosystems, which slows co-design and kernel portability.

Isolated DSL Islands

Emerging MLIR-backed DSLs improve flexibility, but fragmented authoring paths still force teams to choose one stack and lose interoperability.

Core Technology

DSL Translation

Seamlessly mapping high-level intents to hardware-optimized primitives.

LLM-Assisted Optimization

Utilizing generative models to explore the vast kernel search space.

Hardware-Aware Design

Deep integration with next-gen GPU memory hierarchies.

Projects

KernelBridge is grounded in projects that translate compiler and DSL ideas into practical GPU systems outcomes.

FlexAttention

A PyTorch-native attention interface that lets researchers customize FlashAttention for diverse attention variants.

cuFolium

A collaborative framework that bridges Helion and CuTe DSLs, so teams can combine rapid algorithm innovation with expert hardware optimization in one workflow.
